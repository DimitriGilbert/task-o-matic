## EMERGENCY QUICK REFERENCE

**WHEN DISASTER STRIKES:**

### **Immediate Actions (First 5 Minutes)**
1. **Secure your bunker**: Navigate to project directory
2. **Assess damage**: Run `task-o-matic tasks list --status todo`
3. **Establish command**: Use `task-o-matic tasks get-next` to identify critical tasks
4. **Call reinforcements**: Enable AI enhancement with `task-o-matic tasks enhance --task-id <id>`

### **Survival Commands (Memorize These)**
```bash
# Situation assessment
task-o-matic tasks list --status todo
task-o-matic tasks tree

# Critical task management
task-o-matic tasks create "Emergency response" --ai-enhance
task-o-matic tasks split <task-id> --stream
task-o-matic tasks plan <task-id>

# Intelligence gathering
task-o-matic tasks document <task-id> --force
task-o-matic prd parse --file requirements.md --stream

# Complete workflow recovery
task-o-matic workflow --stream
```

### **Contingency Protocols**

**Power Outage**
- Local storage ensures data survival
- All critical data in `.task-o-matic/` directory
- No cloud dependency for core operations

**Network Failure**
- AI operations work with cached models where available
- Documentation fetching may fail, but cached docs remain accessible
- Core task management operates offline

**Team Loss**
- Task documentation enables knowledge transfer
- Enhanced tasks capture all context and requirements
- PRD maintains project vision

**Deadline Apocalypse**
- Workflow automation maintains progress
- Task prioritization guides focus
- Benchmarking identifies optimal AI models for speed
